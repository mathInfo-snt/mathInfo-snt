Description d'une requête HTTP du client

Voici un exemple de requête HTTP :

GET /dossier/index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla Firefox/65.0 (compatible; windows 10)
Accept: text/html

Explications de la première ligne :

  • GET est la méthode utilisée, ici obtenir le fichier index.html dont le chemin est /dossier/.
  • HTTP/1.1 est la version du protocole utilisé.

Explications de la deuxième ligne :

  • Host précède l'hôte qui détient le fichier demandé.

Explications de la troisième ligne :

  • User-Agent est une chaîne donnant des informations sur le client :
  • Mozilla Firefox/65.0 est le navigateur utilisé par le client.
  • Windows 10 est le système d'exploitation du client.

Explications de la quatrième ligne :

  • Accept: text/html est le type de contenu demandé et accepté par le navigateur (ici des pages html).



Description d'une réponse HTTP du serveur

Voici un exemple de réponse HTTP :

HTTP/1.1 200 OK
Date: Sat, 18 Jan 2019 15:36:21 GMT Server: Microsoft-IIS/10.0.17763.1
Content-Type : text/html
Content-Length : 1245

Explications de la première ligne :

  • HTTP/1.1 est la version du protocole utilisé.
  • 200 OK est le code de statut, ici il signifie que la requête s'est accomplie correctement.

Explications de la deuxième ligne :

  • Date précède la date de transfert des données, ici le 18 janvier 2019 à 15h36m12s.
  • Server précède les caractéristiques du serveur avec notamment son adresse IP, ici Microsoft-IIS à l'adresse IP : 10.0.17763.1.

Explications de la troisième ligne :

  • Content-Type donne le type de contenu du corps de la réponse HTTP.

Explications de la quatrième ligne :

  • Content-Length donne la longueur en nombre d'octets, du corps de la réponse HTTP, ici 1245 octets.



Précisions supplémentaires

Quelques exemples de méthodes dans une requête HTTP

  • GET : méthode utilisée pour obtenir des données, par exemple une page html.
  • POST : méthode utilisée pour envoyer des données vers le fichier indiqué, par exemple des identifiants.
  • DELETE : méthode utilisée pour supprimer le fichier indiqué.

Quelques exemples de type de contenu accepté dans une requête HTTP

  • text/html : des pages web avec l'extension .html.
  • image/png : des fichiers images de type png avec l'extension .png.
  • video/mpeg : des fichiers vidéos de type mpeg avec l'extension .mpeg.

Quelques exemples de code de statut dans une réponse HTTP

  • 200 OK : la requête a réussi.
  • 300 Multiple Choice : la requête a plusieurs réponses possibles.
  • 400 Bad Request : le serveur n'a pas pu comprendre la requête à cause d'une syntaxe invalide.
  • 404 Not Found : le serveur n'a pas trouvé le fichier demandé.
