La stéganographie, c'est l'art de dissimuler des données dans d'autres données. Il existe plusieurs techniques différentes qui permettent de faire cela. La stéganographie existe depuis longtemps, bien avant l'invention de l'ordinateur.
Avec l'avénement de l'informatique et le développement des échanges électroniques, les possibilités de cacher un message se sont multipliées : on peut cacher un message dans une image, un site internet, un programme, une musique. La stéganographie a aussi trouvé des applications commerciales, avec le tatouage numérique. On cache un message dans une image ou une musique, pour identifier son origine, et aussi pour empêcher qu'elle ne soit dupliquée à l'insu de son propriétaire.

Exemple de stéganographie informatique

On va voir dans ce paragraphe comment on peut cacher des informations dans une image numérique.
Le principe est basé sur la constitution des images numériques sous formes d'une succession d'octets qui codent les niveaux de rouge, vert et bleu des pixels de l'image.

Octet et bit de poids faible

Un octet est une succession de 8 bits. Un octet permet de coder en binaire tous les nombres entiers de 0 à 255 compris.

Dans un octet on appelle bits de poids faible les bits situés à droite dans l'écriture binaire de l'octet.

Le principe pour cacher des informations dans une image est de cacher les informations dans les bits de poids faible. En effet, modifier les bits de poids faible entraine une légère modification de la valeur décimale :

  • de -1 ou +1 si on modifie seulement le bit le plus à droite ;
  • de -3 à +3 si on modifie les deux bits les plus à droite ;
De telles modifications sont mineurs et ne sont pas perceptibles par l'oeil humain.

Code ASCII

Le code ASCII (American Standard Code for Information Interchange) est un code normalisé. Il associe à chaque caractère un code informatique. Il est utile pour le stockage de texte, l'affichage à l'écran mais aussi pour l'impression. Le code ASCII prévoit au départ 128 caractères numérotés de 0 à 127, codés en binaire sur 7 bits. Par exemple le code ASCII de 'a' est 97, celui de 'A' est 65 et celui de '0' est 48. Cependant, pour répondre aux besoins des langues européennes comprenant des lettres accentuées (comme à, é, ù...) et d'autres symboles (§, µ, £...), le code ASCII a dû être étendu. Le plus souvent, on utilise la norme utf-8.

Exemple de stéganographie sur le bit de poids faible

Repérer le bit de poids le plus faible est très facile sur l'écriture décimale.

  • Si ce bit est égal à 0 il s'agit d'un nombre pair
  • Si ce bit est égal à 1 il s'agit d'un nombre impair
Par exemple, pour 255 (11111111) le bit de poids faible est 1 et pour 254 (11111110) le bit de poids faible est 0.
Ainsi on utilisera ce principe qu'un nombre pair code un 0 et un nombre impair code un 1.

On considère ci-dessous, le début d'un fichier numérique nommé im1.ppm.

 P3
 100  80
 255
 180   165   94   194   101   213   89   134
 ...

Ainsi sur la 4ème ligne, les 8 premières valeurs de niveau de rouge, vert, bleu permettent de coder l'octet suivant : (01001110) qui correspond à 78, c'est à dire au caractère N dans le code ASCII. On a ainsi caché le caractère N dans la 4ème ligne du fichier im1.ppm ci-dessus.
On peut donc utiliser ce principe simple : une valeur paire code 0, une valeur impaire code 1 pour cacher des informations dans une image numérique, quitte à modifier légèrement le fichier image en changeant les bits de poids faible, ce qui est imperceptible à l'oeil humain.

Changer le bit de poids faible : Changer le bit de poids faible, revient à :
  • ajouter 1 pour un entier pair, par exemple : 130 devient 131 ;
  • retrancher 1 pour un entier impair, par exemple : 115 devient 114 .
