Exercices :

Il est possible de classer les formats d’images en deux grandes familles :

  • Les images dites « matricielles » (BitMap en Anglais), qui peuvent représenter des photos, des dessins, des plans etc ... Ces images en couleur ou noir et blanc sont constituées de tous les points (pixels) de l’image. Agrandir ou rétrécir ces images pose certains problèmes afin de déterminer les pixels de la nouvelle image. Par contre, elles sont souvent d’une grande qualité car elles contiennent toute l’information.
    Ces images sont dites matricielles car elles sont enregistrées dans un fichier sous la forme d'une matrice, c'est à dire un tableau de pixels à deux dimensions. Ces images peuvent être compressées à l'aide d'algorithme de compression pour qu'elles prennent moins de place en mémoire.
  • Les images dites « vectorielles », qui sont formées de courbes qui définissent des formes à l’aide d’équations mathématiques. Même pour des formes complexes, les images vectorielles nécessitent moins de place en mémoire que les images BitMap. De plus elles supportent parfaitement les changements d’échelle.

Les images matricielles

Définition d'une matrice

Une matrice est un tableau de nombre à deux dimensions. Sa taille est donnée par deux nombres entiers :

  • le nombre de colonnes
  • le nombre de lignes

Les formats non compressés PBM, PGM et PPM

Le format PBM

Le format PBM (Portable BitMap) est un format dans lequel une image est considérée comme une matrice de pixels noirs ou blancs.
Dans le fichier PBM un pixel noir est codé par un 1 et un pixel blanc est codé par un 0.
Ce type de fichier permet de représenter les caractères du code ASCII.

  • sur la première ligne, on indique le numéro du format PBM qui est P1 ;
  • sur la deuxième ligne, on indique les dimensions de l'image dans l'ordre suivant : nombre de colonnes puis nombre de ligne ;
  • sur les lignes suivantes, on écrit la matrice de 0 et de 1 qui code les pixels blancs et noirs de l'image. Les bits (0 ou 1) sont séparés soit par un espace, soit par un retour à la ligne.

Le format PGM

Le format PGM (Portable GrayMap) est un format dans lequel une image est considérée comme une matrice de pixels gris avec plusieurs niveau de gris allant du blanc au noir.
Dans le fichier PGM le pixel noir est codé 0 et le pixel blanc est codé par une valeur comprise entre 0 et 255 (dans l'exemple ci-dessous le blanc est codé 15). Tous les codes intermédiaires codent des niveaux de gris (plus le code est élevé plus le gris est clair).

  • sur la première ligne, on indique le numéro du format PGM qui est P2 ;
  • sur la deuxième ligne, on indique les dimensions de l'image dans l'ordre suivant : nombre de colonnes puis nombre de lignes ;
  • sur la troisième ligne, on indique la valeur maximum de niveau de gris comprise entre 0 et 255. Cette valeur maximum code le blanc ;
  • sur les lignes suivantes, on écrit la matrice des valeurs qui codent les niveaux de gris de chaque pixels.

Le format PPM

Le format PPM (Portable PixMap) est un format dans lequel une image est considérée comme une matrice de pixels, chacun d'eux étant codé en RVB, donc par trois valeurs donnant l'intensité de chaque couleur primaire : rouge, vert, bleue.
Par exemple dans le fichier PPM, un pixel rouge est codé :   255 0 0   un pixel vert est codé :   0 255 0   et un pixel bleu est codé :   0 0 255 .

Ce fichier PPM est constitué de la façon suivante :

  • sur la première ligne, on indique le numéro du format PPM qui est P3 ;
  • sur la deuxième ligne, on indique les dimensions de l'image dans l'ordre suivant : nombre de colonnes puis nombre de lignes ;
  • sur la troisième ligne, on indique la valeur maximum comprise entre 0 et 255 de chaque couleur primaire (souvent égale à 255) ;
  • sur les lignes suivantes, on écrit la matrice des triplets de valeurs, chacun de ces triplets donnant le code RVB des pixels de l'image (par exemple le triplet : 255 0 0 code la couleur rouge).

Les formats compressés

Les formats d’image représentant l’ensemble des pixels sous forme d'une matrice sont très volumineux. Il existe plusieurs algorithmes de compression de données qui réduisent la taille des fichiers. Il y a deux grandes familles : les méthodes de compression d’image sans perte et celles avec perte.

Les formats compressés les plus utilisés

Les formats compressés les plus utilisés sont : TIFF, GIF, JPEG ou JPG, et PNG.

  • TIFF (Tagged Image File Format) : Ce format est orienté vers les professionnels (imprimeurs, publicitaires...). Il permet d’obtenir une image de très bonne qualité, mais sa taille reste volumineuse. Ces fichiers TIFF peuvent être compressés (compression LZW).
  • GIF (Graphics Interchange Format) : Ce format est un des standards d’internet. Les fichiers sont de petite taille, car les images ne contiennent que 256 couleurs. Ce format permet d’avoir des parties d’image transparentes.
  • JPEG (Joint Photographic Expert Group) : Ce format offre plusieurs taux de compressions, qui affectent la qualité de l’image. Il est devenu le standard des formats d’image sur internet, car on obtient des images plus petites grâce à la compression. Ceci permet de les charger rapidement, même avec une connexion bas débit.
  • PNG : (Portable Network Graphic) : C’est le futur standard pour internet, car il autorise la transparence comme le format GIF et autorise 16.7 millions de couleurs. Il permet aussi la compression de données.

Les compressions sans perte

Les méthodes les plus importantes de compression d’image sans perte sont :

  • la méthode du codage des répétitions (comme LRE), utilisée sur les premiers scanners et télécopieurs.
  • le codage par quadtree qui utilise une représentation sous forme d'arbre dans lequel chaque sommet à quatre arêtes ;
  • le codage entropique (comme le code de Huffman) qui utilise un arbre construit à partir de la fréquence de chaque caractère de l'image de départ. Le but est de coder les caractères plus fréquents en utilisant moins de place que les caractères moins fréquent à l'aide d'un dictionnaire qui dépends des fréquences de chaque caractère.
  • les algorithmes à dictionnaire adaptable tels que LZW. L’idée de cet algorithme est de construire une table de traduction à partir des informations à compresser.

Exemple de compression LRE

On va appliquer le principe du codage des répétitions (LRE) à un fichier image au format PBM.
Le principe est de remplacer toutes suites successives de 0 ou de 1 par le nombre de 0 ou de 1.
Par exemple :

  • 0 0 0 0 0 sera remplacée par : 6 0
  • 1 1 1 sera remplacée par : 3 1
  • 0 0 0 0
    0 0 1 1
    sera remplacée par :
    6 0
    2 1

Les compressions avec perte

Il existe de nombreuses techniques de compression avec perte. Le format JPEG utilise un processus de compression et de décompression qui comporte six étapes principales. Ce processus est long et compliqué à expliquer. Il s'agit d'une compression irréversible, puisqu’il y a perte de données. Comme il y a des pertes de données, on peut obtenir un taux de compression plus important.

Les images vectorielles

Il existe de nombreux formats de fichiers vectoriels, comme Postscript, PDF, Adobe Flash, Illustrator ou SVG. Aujourd’hui le format standard des images vectorielles est un format PostScript appelé l’EPSF (Encapsulated PostScript File) ce qui signifie PostScript encapsulé. Son extension est .eps ou .epsf. Les images vectorielles sont moins volumineuses que les images matricielles et subissent sans déformation des changements d’échelle.
